🇧 English version

General Information

The "AntiMax" browser extension is designed for local encryption and decryption of text messages in supported web services.

The main purpose of the extension is to allow users to transform text messages between plain text and encrypted format directly inside their browser.

Data Processing

The extension may access text content that the user chooses to encrypt or decrypt solely for the purpose of performing the extension's core functionality.

All data processing is performed locally on the user's device within the browser.

The extension:

  • does not send message content to remote servers;
  • does not transfer user data to third parties;
  • does not store message content;
  • does not create a history of processed messages;
  • does not use processed data for advertising, analytics, tracking, or user profiling.

Text data is processed only while the user performs an encryption or decryption operation. The processed content remains on the user's device and is not transmitted outside the browser.

🌐 Use of Page Content Access

The extension uses access to supported websites only to integrate encryption and decryption functionality directly into website interfaces.

The extension may access and modify message text only when required to perform an action initiated by the user.

The accessed text is used exclusively for:

  • converting plain text messages into encrypted format;
  • converting encrypted messages back into readable text;
  • inserting the processed result back into the user's interface.

The extension does not collect, analyze, classify, or monitor user conversations. It does not use message content for purposes unrelated to encryption or decryption.

Clipboard Access

The extension uses clipboard access to provide encryption and decryption functionality.

The clipboardRead permission is required to retrieve text that the user has copied for processing inside the extension popup window.

Clipboard access is used only when the user performs an action such as:

  • encrypting copied text;
  • decrypting copied encrypted text;
  • processing text inside the extension popup.

The extension does not automatically read the clipboard in the background and does not access clipboard contents without a user-initiated action.

The clipboardWrite permission is used to copy the encryption or decryption result back to the user's clipboard when requested by the user.

💾 Data Storage

The extension does not store message content, encryption keys, or processed text on external servers.

If extension settings are used, they may be stored locally using the standard Chrome storage API (chrome.storage) and are not transmitted to the developer or any third party.

🤝 Data Sharing

The extension does not share user data with third parties.

The extension does not use:

  • external services for message processing;
  • remote servers for encryption or decryption;
  • analytics systems;
  • advertising or tracking technologies.

All encryption and decryption operations are performed locally in the user's browser.

🔒 Security

The extension is designed to minimize external data processing.

Encryption and decryption are performed directly on the user's device without sending message content to external systems.

The user remains in control of all data processing actions. The extension only performs operations initiated through the extension interface or user interaction.

Общая информация

Расширение «АнтиМакс» предназначено для локального шифрования и расшифрования текстовых сообщений в поддерживаемых веб-сервисах.

Основная задача расширения — предоставить пользователю возможность преобразовывать текстовые сообщения между открытым и зашифрованным видом непосредственно в браузере.

🔐 Обработка данных

Расширение может получать доступ к тексту, который пользователь выбирает для шифрования или расшифрования, исключительно для выполнения функций расширения.

Обработка данных выполняется локально на устройстве пользователя внутри браузера.

Расширение:

  • не отправляет содержимое сообщений на удаленные серверы;
  • не передает данные третьим лицам;
  • не сохраняет содержимое сообщений;
  • не создает историю обработанных сообщений;
  • не использует данные для рекламы, аналитики или профилирования пользователей.

Текстовые данные используются только во время выполнения операции шифрования или расшифрования и после завершения обработки не передаются за пределы устройства пользователя.

🌐 Использование доступа к содержимому страниц

Расширение использует доступ к поддерживаемым веб-сайтам только для интеграции функций шифрования и расшифрования непосредственно в интерфейс сайтов.

Расширение может получать и изменять текст сообщений только в рамках действий, выполняемых пользователем.

Полученный текст используется исключительно для:

  • преобразования обычного текста в зашифрованный;
  • преобразования зашифрованного текста обратно в исходный вид;
  • вставки результата обработки обратно в интерфейс пользователя.

Расширение не выполняет анализ содержимого сообщений, классификацию данных, отслеживание переписки или сбор информации о действиях пользователя.

Использование буфера обмена

Для работы функций шифрования и расшифрования используется доступ к буферу обмена браузера.

Разрешение на чтение буфера обмена (clipboardRead) необходимо для получения текста, который пользователь самостоятельно копирует для дальнейшей обработки в окне расширения.

Доступ к буферу обмена используется только:

  • при выполнении пользователем операции шифрования;
  • при выполнении пользователем операции расшифрования;
  • для обработки текста внутри popup-окна расширения.

Расширение не выполняет автоматическое чтение буфера обмена в фоновом режиме и не получает доступ к содержимому буфера без действия пользователя.

Разрешение на запись в буфер обмена (clipboardWrite) используется для копирования результата шифрования или расшифрования по запросу пользователя.

💾 Хранение данных

Расширение не хранит содержимое сообщений, ключи шифрования или результаты обработки на внешних серверах.

Если используются настройки расширения, они могут сохраняться только локально через стандартное хранилище браузера Chrome (chrome.storage) и не передаются разработчику.

Передача данных третьим лицам

Расширение не передает пользовательские данные третьим лицам.

Расширение не использует:

  • сторонние API для обработки сообщений;
  • серверы для выполнения шифрования или расшифрования;
  • системы аналитики;
  • рекламные или отслеживающие технологии.

Все операции выполняются локально в браузере пользователя.

🔒 Безопасность

Архитектура расширения разработана с учетом минимизации обработки пользовательских данных.

Шифрование и расшифрование выполняются непосредственно на устройстве пользователя без передачи содержимого сообщений в сеть.

Пользователь самостоятельно инициирует все операции обработки текста через интерфейс расширения.
